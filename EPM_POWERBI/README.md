# Oracle EPM to Power BI Connector Generator

## Overview
This repository contains the **Oracle EPM to Power BI Connector Generator**, a standalone Windows application designed to automatically build highly secure, custom data extraction pipelines linking Oracle EPM Cloud (Enterprise Performance Management) directly to Microsoft Power BI.

This product allows Administrators and Developers to generate custom connection scripts using a simple Graphical User Interface (GUI), without writing a single line of M-Query or Python code themselves. The generated scripts handle authenticating against Oracle EPM, fetching specialized Data Slices via REST API, structuring the raw JSON into flat tables, and directly importing the Pandas DataFrame into Power BI.

The Generator outputs dual integration architectures, ensuring compatibility across all Power BI environments:
1. **Python / Hybrid M-Query Payload** (Powerful, handles massive datasets, requires local Python).
2. **Native pure M-Query Payload** (Zero-dependency, uses built-in Power Query Web API).

---

## 1. What This Tool Does
When you run the Generator Tool, you will be prompted to enter your EPM Environment details (URL, Username, Password, Cube, Application, etc) and your desired Dimension Grids (POV, Rows, Columns). 

Upon clicking "Generate", the tool creates two output files:
- `power_query_engine.m` - An advanced M-Query template designed to be copy-pasted directly into the Power BI Advanced Editor. It establishes parameters and executes the secure Python backend.
- `custom_epm_pbi_connector.py` - The raw, secure Python engine file for manual execution outside of Power BI or for debugging.

The primary engine generated is uniquely hashed in Base64 encoding. It incorporates a proprietary trial/licensing structure to protect the script's distribution.

## 2. Prerequisites & Installation

### A. To Run the Generator GUI (`generate_pbi_connector_gui.exe`)
Because the Generator itself is a standalone, fully-compiled Windows executable, there are absolutely **ZERO** software prerequisites to run it.
- **Operating System:** Windows 10/11
- **Installation:** Download and double-click `Generator_Tool\generate_pbi_connector_gui.exe`. No Python installations, Administrator privileges, or dependencies are required.

### B. To Run the Generated Python Route inside Power BI
If an end-user wishes to use the `power_query_engine.m` script generated by the tool, their specific local Power BI environment **must** meet the following requirements:
1. **Python Installation:** Python 3.8+ must be installed on the user's machine.
2. **Power BI Configuration:** Power BI Desktop must be configured to point to the local Python installation path (`File > Options and settings > Options > Python scripting`).
3. **Python Libraries:** The generated script will automatically attempt to install the required `pandas` and `requests` packages if they are missing. However, if the user operates behind a corporate proxy or firewall, they must install them manually:
   `pip install pandas requests`

### C. To Use the Native M-Query Route (Zero Python Bypass)
If an end-user operates in an environment where Python **cannot** be installed (or if they prefer a strictly native Power Query experience), they should bypass the Generator entirely and use the exact file located here:
- **`Native_M_Query_Engine/pure_m_query_engine.m`**

This script relies entirely on Power BI's internal `Web.Contents` engine. 
- **Requirements:** None. It works out-of-the-box on any standard PC running Power BI Desktop.

## 3. Quick Start Guide
1. Launch `Generator_Tool\generate_pbi_connector_gui.exe`.
2. Input your Oracle EPM Cloud credentials and the URL domain.
3. Configure your desired target Cube and Application Name.
4. Input your Row, Column, and POV configurations exactly as they appear in EPM (Comma-separated dimensions, Pipe-separated values for members).
5. Click **Generate Power BI Connector Script**.
6. Select an empty folder on your desktop to export your files.
7. Open Power BI Desktop, navigate to **Get Data -> Blank Query**, open the **Advanced Editor**, and paste the contents of the generated `power_query_engine.m` file.

For detailed, step-by-step instructions on the dual architectures, please refer to the documents in the `Documentation/` folder.
